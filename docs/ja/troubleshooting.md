# トラブルシューティング

## 「Kindle is not running」と表示される

Kindle for Mac が起動していないか、プロセス名が認識されていません。

- Mac App Store から Kindle for Mac を開いてください。
- ライブラリや Store ではなく、本のページに移動してください。
- 再度 `kpc` を実行してください。

## 「No suitable reading window was found」と表示される

Kindle は起動しているが、適切なウィンドウが検出されませんでした。

- 本が開いており、読書ウィンドウが表示されている（最小化されていない）ことを確認してください。
- Kindle ウィンドウのサイズが 800x600 ピクセル以上であることを確認してください。
- ウィンドウを大きくしてから再試行してください。

## 「The Kindle window does not appear to show a content page」と表示される

キャプチャしたウィンドウが、ライブラリ・Store・ローディング画面に見えると判断されました。

- 本の最初のページに移動してください。
- ページが完全に読み込まれてから `kpc` を実行してください。

## 「Accessibility permission is required」と表示される

キーイベント送信の権限がありません。

- [パーミッション ガイド](permissions.md) の手順に従って設定してください。
- 権限付与後、ターミナルを再起動してから再試行してください。

## キャプチャ画像が真っ黒になる

画面収録権限がありません。

- [パーミッション ガイド](permissions.md) の手順で権限を付与し、ターミナルを再起動してください。

## ページが進まない

- アクセシビリティ権限がないか、Kindle がフォーカスを失っています。
- `--start-delay` を増やして、キャプチャ開始前に Kindle にフォーカスを移す時間を確保してください。

## PDF にタイトルバーや矢印が混入する

キャプチャ中にマウスが Kindle ウィンドウ上にあると UI が表示されます。

- キャプチャ中はマウスを別の場所に置いてください。
- Kindle をフルスクリーンモードにすることも有効です。
- `failed_pages.json` を確認し、問題のページを `--retry-failed` で再取得してください。

## ページ順が乱れる

- `cropped/` に以前の実行分のファイルが残っている場合があります。
- `cropped/` ディレクトリを削除して再実行するか、`--retry-failed` で失敗ページのみ補完してください。

## 最終ページで止まらない

- `--max-pages` で上限を設定してください。
- 末尾検出は同一フレームの連続 3 回で機能しますが、`--max-pages` を設定しておくと確実です。

## OCR が失敗する

OCR の失敗は致命的ではありません。通常の `book.pdf` は必ず生成されます。

- `tesseract` と `ghostscript` がインストールされているか確認してください。
  ```bash
  which tesseract && which gs
  ```
- `ocrmypdf` を直接実行してエラー詳細を確認することもできます。
- `--debug` フラグを付けると、終了コードを含む詳細なログが表示されます。
