# macOS パーミッション設定

kindle-pdf-capture を実行するには、使用しているターミナルアプリ（Terminal.app、iTerm2 など）に 2 つの権限を付与する必要があります。

## 画面収録（Screen Recording）

ウィンドウの内容をキャプチャするために必要です。

1. **システム設定** を開く
2. **プライバシーとセキュリティ** → **画面収録** を選択
3. `+` ボタンをクリックし、ターミナルアプリを追加
4. チェックボックスを有効にする
5. ターミナルを再起動する

### この権限がない場合の症状

- キャプチャ画像が真っ黒になる（真っ黒な JPEG が生成される）

## アクセシビリティ（Accessibility）

Kindle に右矢印キーイベント（ページ送り）を送るために必要です。

1. **システム設定** を開く
2. **プライバシーとセキュリティ** → **アクセシビリティ** を選択
3. `+` ボタンをクリックし、ターミナルアプリを追加
4. チェックボックスを有効にする

### この権限がない場合の症状

- 「Accessibility permission is required」というエラーが表示される
- ページが自動的に進まない

## 確認方法

以下を実行して、両方の権限が正しく設定されているか確認します。

```bash
kpc --out /tmp/test --max-pages 1 --start-delay 0
```

権限が不足している場合は、具体的な指示が含まれたエラーメッセージが表示されます。

## 注意事項

- 権限はターミナルアプリ単位で付与されます。仮想環境や別のシェルを使っていても、親となるターミナルアプリに権限が必要です。
- 権限を付与した後、変更を反映させるためにターミナルを再起動する必要がある場合があります。
- Apple Silicon Mac でも `sudo` で実行したプロセスに権限は引き継がれません。ターミナルアプリ自体に権限を付与してください。
